# 02 - 选择与迁移路径

> 先确认现有运行时的唯一写入源和唯一调度源，再引入任何记忆或演化组件。默认先只读 / dry-run；安装、配置写入、长期循环和重启都应有明确回滚点。

## 选择决策树

```text
已有 Celia / lossless-claw / 其他结构化记忆？
  ├─ 是：默认不再启用第二条长期记忆写入链
  │     └─ 仅评估检索或可视化能力，确认唯一写入源后再考虑集成
  └─ 否：需要本地、可查询的结构化记忆？
        └─ 评估 MemOS Local；先检查数据库位置、模型调用和卸载方式

需要把错误与反馈沉淀为规则？
  ├─ 需要人工批准的经验闭环：SelfLearn
  └─ 需要演化对象、信号扫描和审计建议：EvoMap（优先 --review，不跑 --loop）

需要定时整理或巡航？
  └─ 为一个目标保留一个调度源；写清停止命令、日志位置、告警和回滚方式
```

## 组件边界

| 组件 | 可以做 | 不应假定 | 上线前检查 |
|---|---|---|---|
| MemOS Local | 本地记忆存储、检索、可视化 | 不自动保证记忆正确或与现有系统无冲突 | 写入位置、唯一写入源、模型/网络依赖、卸载方案 |
| EvoMap | 生成演化建议与审计对象 | 不会可靠地自动修复生产系统 | `--review` 输出、联网行为、停止方式、提交前人工审查 |
| Self-Improving Agent | 将经验整理为候选规则/技能 | 不替代测试、安全审批或代码评审 | 事件来源、人工 promotion、最小验证证据 |
| SelfLearn | open → resolved → draft → GEP → promotion 闭环 | 不自动批准、不自动写永久规则 | promotion 目标、审核记录、回滚点 |

## 安装、验证、停用、回滚

1. **安装前**：记录当前启用的 memory、cron、插件和配置快照；不要在生产记忆上试新写入链。
2. **验证**：使用临时工作区或 dry-run，确认输入、输出、写入位置、联网行为和费用。
3. **停用**：先停止对应 cron / loop / 插件，再确认没有后台写入或排队任务。
4. **回滚**：恢复配置快照；只在确认数据格式和来源后处理新增数据库或摘要，禁止盲删。

## 组合使用的最低规则

- 一类长期记忆只有一个主写入源。
- 一个任务目标只有一个周期调度源。
- 任何自动 promotion、外发、安装或重启都保持人工确认。
- 冲突或不确定时，停止写入，保留证据并回到只读模式。
